Este documento tiene el propósito de explicar aspectos técnico necesario para que el aplicativo funcione de manera óptima. La configuración de IIS y sus versiones, Base datos, y la configuración de Reporting Services. Esto acompañado de Cuentas de servicios necesarias para la interacción de sus componentes.

2.	Requerimientos del Sistema
2.1.	Requerimientos de Hardware
Para el funcionamiento óptimo de la aplicación SMIS serán en 2 Servidores un para Internet Information Services (IIS) y otro para la Base de datos Sql Server Management v17.1 junto con Microsoft SQL Server Reporting Services por lo tanto será necesario como requisito: 

	Disco Duro: 100GB
	Memoria RAM: 32GB

2.2.	Requerimientos de Software
Requisito de software para la aplicación: 

	Sistema Operativo: Windows Server 2016 
	Motor Web: Internet Information Services (IIS) 8 o superior 
	Framework: Microsoft .NET 4.5 de 64 bits 
	Base de datos: SQL Server 
3.	Instalación del Software
Como requisito previo para la instalación del sistema debe estar configurado:
	Internet Information Services 8.5. con FrameWork 4.5
	SQL Server 2016
	Reporting Services 2016

Se le enviara al administrador los siguientes componentes:
	Script de la base de datos 
	Carpeta con los archivos de reportes y fuentes de datos (data sources)
	Carpeta con los archivos pre-compilados de la aplicación ASP.NET 

Paso para instalación de la aplicación SMIS
1.	Ejecutar los scripts de la base de datos 
2.	Subir a Reporting Services los archivos con reportes y fuentes de datos (data sources)
3.	Cambiar la conexión de la base de datos de la fuente de datos (data sources)
4.	Crear un Sitio Web a Internet Information Services con los archivo de Centralizador Contable de la aplicación ASP.NET.

3.1.	Base de datos 
El sistema requiere una base de datos instalada en el servidor de Base de Datos de Producción (Se dieron los requerimientos en el punto 2.1 y 2.2). Los scripts para la creación del esquema y poblamiento de la base de datos están diseñados para trabajar con Microsoft SQL Server Management v17.1.
La Base de Datos debe llamarse:




3.1.1.	Script de generación de esquema
Para el correcto despliegue de las bases de datos del aplicativo Centralizador Contable, se debe ejecutar la script de la creación de base de datos:
1.	Creación de base de datos: 01_Create_DB_SCC.sql
2.	El esquema será generado a través de un archivo Backup: 02_Schema_BD_SCC.sql]
3.1.2.	 Script de poblamiento de tablas
1.	El poblamiento de las Tablas será a través de un archivo Backup: BD_SCC.bak
a.	Archivo backup contendrá toda la información ya analizada y trabajada con los usuarios finales. 
2.  Se tiene que ejecutar el siguiente script: 03_Limpieza_DataEsquemaSCC.sql

3.1.3.	Cuentas y permisos 
Se tiene que crear el usuario SCCUser siendo propietario (owner) de la base de datos BD_SCC en producción. Se tiene que configurar la nueva clave en Producción y configurar en el Web.Config del archivo ASP.NET.
3.2.	Sitio Web IIS
Para el correcto despliegue del front-end de la Aplicación SMIS, se debe realizar los siguientes pasos en Internet Information Services (IIS).

Modo de canalización administrada: Integrated

1.	Propósito

Crear una plataforma que pueda centralizar toda la información de Becas y Cae, Control de ingreso e incobrables,  integrando los procesos financieros. Finalmente, pueda ser una herramienta para la centralización  de la información contable. Por lo cual, debe ser capaz de recibir información desde Matriculas, FUA y fin700.

2.	Descripción del Sistema
SCC agrupa los procesos de Becas y CAE, Cierre Comercial e Incobrables, con un fin operativo se pretende lograr el máximo detalle de la información para saber la situación actual y futura.

2.10.	Beca y Cae - Carga Imputación paso a paso 
Aquí podremos generar imputaciones de beneficios mediante cargas de archivos del tipo csv y txt. Con esto automáticamente nos pedirá los parámetros de año y mes de imputación para su proceso.

 

Luego de seleccionar el año y el periodo , se procede a la carga de archivos 

   

Una vez cargados los archivos, debemos presionar botón “guardar” y presionar el botón “siguiente” que se encuentra más abajo.

 


Ahora nos encontramos en la ficha Cierre comercial, presionamos guardar y botón siguiente.
 
 



Al procesar cierre comercial pasaremos a la ficha Saldos vigentes presionamos guardar y botón siguiente.

 

Al procesar Saldos Vigentes se ubicara la ficha en  Ctrl. Ingresos, damos click en botón guardar y luego presionamos botón siguiente. 


 
 
Al procesar Ctrl de ingresos se ubicara la ficha en FUC, damos click en botón guardar y luego presionamos botón siguiente. 

 

Al procesar FUC se ubicara la ficha en Sus. Beca, damos click en botón guardar y luego presionamos botón siguiente. 
                                                        
 


Al procesar Sup. Beca esta por ultimo queda en la ficha Pagos Estudiantes, damos clic en botón guardar y siguiente. Con esto concluirá el proceso de BECA Y CAE exitosamente.-

2.26.	Incobrables – Carga – Incobrabilidad (CI)
El proceso de carga de provisión consta de 4 etapas :  
2.26.1.	(CI)  Carga Provisiones.
La primera etapa consiste en la carga de provisión del Periodo anterior y actual, con el fin de comparar los resultados. (agregar cuadro ) 

 
2.26.2.	(CI)  Movimientos contra Provisión.
La segunda etapa se cargan todos los movimientos contables relacionados a la cuenta contable de provisión según periodo seleccionado. (agregar cuadro)

 
2.26.3.	(CI) Control de Ingresos.
La tercera etapa consiste en la  carga de todos Las carreras de los estudiantes según previa carga de provisión.

 


2.26.4.	(CI) Procesar
En la cuarta y última etapa se realiza calculo del incobrables mostrando su resultado por pantalla.

 
2.27.	Incobrables – Carga – Provisión (CP)
Modulo en cual se realiza el calculo de provisión de incobrables. Contsa de 8 etapas : 
2.27.1.	(CP) Selección Periodo Proceso
En la primera etapa se debe seleccionar el año y el més del periodo a calcular.

 
2.27.2.	(CP) Saldos Vigentes
La segunda etapa consiste en la carga de datos de los saldos vigentes según cuentas contables cargadas para el incobrables.

 

 
Al terminar el proceso se desplegará un mensaje.

 


2.27.3.	(CP) Calculo Becas y Cae 
La tercera etapa consist en la carga de los beneficios de Becas & CAE por estudiantes según la carga de Saldos Vigentes.

 

 
2.27.4.	(CP) Calculo Acelerados
La cuarta etapa consiste en la carga de los estudiantes donde se les hallan acelerado su pago todo esto según la carga de Saldos Vigentes.

 
2.27.5.	(CP) Calculo Deserciones 
La quinta etapa consiste en la carga desde el sistema FUC de los Desertores Formales para el proceso de Provisión. 

 




2.27.6.	(CP) Reconocimiento de Ingreso
La sexta etapa consiste en la carga el reconocimiento de ingreso según periodo seleccionado para la provisión.

 





2.27.7.	 (CP) Calculo Repactados 
La septima etapa consiste en la carga de repactados.

 

2.27.8.	(CP) Procesar
La octava etapa consiste en la corroboración de los datos ingresados, y posteriormente en la ejecución  del proceso de provisión para el incobrable.
